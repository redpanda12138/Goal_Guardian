# 数据库表创建问题修复

## 问题分析

### 错误信息
```
sqlalchemy.exc.ProgrammingError: (pymysql.err.ProgrammingError) (1146, "Table 'goalguardian_db.patient_mapping' doesn't exist")
```

### 根本原因

**问题：** `patient_mapping` 表不存在

**原因：** `app/db/mas_entities.py` 文件缺少表创建代码

**对比其他实体文件：**
- `account_entities.py` ✅ 有 `Base.metadata.create_all(engine)`
- `topic_entities.py` ✅ 有 `Base.metadata.create_all(engine)`
- `chat_entities.py` ✅ 有 `Base.metadata.create_all(engine)`
- `sys_entities.py` ✅ 有 `Base.metadata.create_all(engine)`
- `mas_entities.py` ❌ **缺少** `Base.metadata.create_all(engine)`

## 已实施的修复

在 `app/db/mas_entities.py` 文件末尾添加了：
```python
# 创建表（如果不存在）
Base.metadata.create_all(engine)
```

## 修复步骤

### 步骤1：重启服务器

重启服务器后，`mas_entities.py` 会被导入，`Base.metadata.create_all(engine)` 会自动创建 `patient_mapping` 表。

### 步骤2：验证表已创建

**方法1：检查数据库**
```sql
USE goalguardian_db;
SHOW TABLES LIKE 'patient_mapping';
DESCRIBE patient_mapping;
```

**方法2：查看服务器日志**
重启后应该没有表不存在的错误。

**方法3：测试API**
尝试创建MAS会话，应该不再出现表不存在的错误。

## 表结构

`patient_mapping` 表结构：
- `id` (String(80), PRIMARY KEY)
- `account_id` (String(80), UNIQUE, NOT NULL)
- `patient_id` (String(80), UNIQUE, NOT NULL)
- `created_at` (DateTime)
- `updated_at` (DateTime)
- 索引：
  - `idx_patient_mapping_account_id` on `account_id`
  - `idx_patient_mapping_patient_id` on `patient_id`

## 注意事项

1. **表创建时机：** 表会在服务器启动时自动创建（当 `mas_entities.py` 被导入时）
2. **幂等性：** `create_all()` 是幂等的，如果表已存在不会报错
3. **索引：** 表创建时会自动创建定义的索引
