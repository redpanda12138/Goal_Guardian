# MAS系统与talkieai-server整合方案分析

## 整合目标

将MAS系统的健康教练会话管理功能整合到talkieai-server后端中，实现：
1. 统一的API接口
2. 共享的数据库和用户系统
3. 统一的AI服务管理
4. 保持MAS系统的核心功能不变

## 整合策略

### 方案一：完全整合（推荐）
**将MAS的6个代理作为talkieai-server的内部服务模块**

**优点**:
- 统一的数据管理（使用数据库替代JSON文件）
- 统一的用户认证和授权
- 共享AI服务配置
- 简化部署（单一服务）
- 更好的错误处理和日志管理

**缺点**:
- 需要重构MAS的代码结构
- 失去微服务的独立扩展能力

**实施步骤**:
1. 将MAS代理转换为talkieai-server的服务模块
2. 将JSON文件存储迁移到数据库
3. 集成用户系统（patient_id映射到account_id）
4. 统一AI服务调用
5. 创建新的API路由

---

### 方案二：混合架构
**保持MAS作为独立服务，talkieai-server作为网关**

**优点**:
- 保持MAS系统的独立性
- 最小化代码改动
- 可以独立扩展MAS服务

**缺点**:
- 需要维护两套系统
- 数据同步复杂
- 部署复杂度增加

**实施步骤**:
1. talkieai-server作为API网关
2. 在talkieai-server中创建代理层，转发请求到MAS服务
3. 实现数据同步机制（数据库 ↔ JSON文件）
4. 统一用户认证

---

### 方案三：渐进式整合
**先整合数据层，再逐步整合服务层**

**优点**:
- 风险较低，可以逐步验证
- 保持向后兼容

**缺点**:
- 整合周期较长
- 中间状态可能复杂

---

## 推荐方案：方案一（完全整合）

### 1. 数据库设计

```sql
-- 患者信息表（扩展account表或新建）
CREATE TABLE patient_info (
    id VARCHAR PRIMARY KEY,
    account_id VARCHAR REFERENCES account(id),
    preferred_name VARCHAR,
    hobbies TEXT,  -- JSON数组
    family TEXT,   -- JSON数组
    friends TEXT,  -- JSON数组
    travel TEXT,   -- JSON数组
    created_at TIMESTAMP,
    updated_at TIMESTAMP

-- SMART目标表
CREATE TABLE smart_goals (
    id VARCHAR PRIMARY KEY,
    patient_id VARCHAR REFERENCES patient_info(id),
    goal_text TEXT,
    date DATE,
    status VARCHAR,  -- active, completed, archived
    created_at TIMESTAMP
);

-- 会话预约表
CREATE TABLE review_schedules (
    id VARCHAR PRIMARY KEY,
    patient_id VARCHAR REFERENCES patient_info(id),
    next_review_time TIMESTAMP,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 健康教练会话记录表
CREATE TABLE health_coach_sessions (
    id VARCHAR PRIMARY KEY,
    patient_id VARCHAR REFERENCES patient_info(id),
    session_type VARCHAR,  -- opening, goal_review, closing

    turn_index INTEGER,
    chat_history TEXT,      -- JSON数组
    status VARCHAR,         -- active, completed
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 会话摘要表
CREATE TABLE session_summaries (
    id VARCHAR PRIMARY KEY,
    session_id VARCHAR REFERENCES health_coach_sessions(id),
```

### 5. 定时任务实现

使用APScheduler替代OA的后台线程：

```python
# app/services/mas/orchestration_service.py
from apscheduler.schedulers.background import BackgroundScheduler

class OrchestrationService:
    def __init__(self, db: Session):
        self.db = db
        self.scheduler = BackgroundScheduler()
        self.scheduler.add_job(
            self.check_review_schedules,
            'cron',
            minute=0  # 每小时整点执行
        )
        self.scheduler.add_job(
            self.extract_daily_notes,
            'cron',
            hour=0, minute=0  # 每天午夜执行
        )
        self.scheduler.start()
```

### 6. 用户系统集成

将MAS的`patient_id`映射到talkieai-server的`account_id`：

```python
# 在AccountService中添加
def get_or_create_patient_info(self, account_id: str):
    """获取或创建患者信息"""
    patient = self.db.query(PatientInfo).filter_by(account_id=account_id).first()
    if not patient:
        patient = PatientInfo(
            id=f"patient_{short_uuid()}",
            account_id=account_id
        )
        self.db.add(patient)
        self.db.commit()
    return patient
```
