# MAS对话记录和查看指南

## 一、对话记录机制

### ✅ MAS系统会自动记录所有对话

MAS系统在以下位置记录对话：

1. **OA服务** (`memory/goal_reviews.json`) - **主要记录位置**
   - 记录所有代理的完整对话历史
   - 包含所有轮次（turn_index）
   - 格式：`[{"patient_id": "...", "turn_index": 1, "chat_history": [...]}]`

2. **各代理服务** - 各自维护会话记录
   - SOA: `memory/soa_conversations.json`
   - GRA: `memory/gra_conversations.json`
   - SCA: `memory/sca_conversations.json`
   - SSA: `memory/session_summaries.json`

### 记录流程

```
用户发送消息
    ↓
talkieai-server转发到MAS服务
    ↓
MAS服务处理并生成回复
    ↓
OA服务保存到goal_reviews.json (完整历史)
    ↓
各代理服务保存到各自的conversations.json (代理特定历史)
```

## 二、如何触发对话

### 方式1：手动触发（推荐用于测试）

#### 步骤1：触发会话开始

```bash
curl -X POST "http://localhost:8097/api/v1/mas/sessions/trigger" \
  -H "X-Token: {your_jwt_token}" \
  -H "Content-Type: application/json" \
  -d '{}'
```

这会触发SOA（Session Opening Agent），开始新的健康教练会话。

**响应示例**:
```json
{
  "code": "200",
  "status": "SUCCESS",
  "message": "success",
  "data": {
    "status": "SOA triggered",
    "patient_id": "patient_xyz789"
  }
}
```

#### 步骤2：发送消息

```bash
curl -X POST "http://localhost:8097/api/v1/mas/sessions/message" \
  -H "X-Token: {your_jwt_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "user_input": "我今天的能量水平是7分",
    "turn_index": 2
  }'
```

**响应示例**:
```json
{
  "code": "200",
  "status": "SUCCESS",
  "message": "success",
  "data": {
    "status": "message processed",
    "turn_index": 3
  }
}
```

### 方式2：定时自动触发

OA服务会每小时检查一次，如果到达预约时间，会自动触发会话。

**预约时间设置**:
- 通过提交会话笔记时自动设置
- 或通过创建预约API设置

## 四、完整对话流程示例

### 1. 触发会话

```bash
POST /api/v1/mas/sessions/trigger
```

**系统行为**:
- SOA被触发
- 从MMA获取患者信息
- 生成开场问候
- 保存到 `soa_conversations.json` 和 `goal_reviews.json`

### 2. 发送消息（Turn 2）

```bash
POST /api/v1/mas/sessions/message
{
  "user_input": "我今天的能量水平是7分",
  "turn_index": 2
}
```

**系统行为**:
- SOA处理消息
- 生成回复
- 更新 `goal_reviews.json` 中的对话历史

### 3. 继续对话（Turn 3-6）

重复步骤2，每次增加 `turn_index`

### 4. 自动切换到GRA（Turn 7）

当 `turn_index` 达到6时，SOA会自动触发GRA

### 5. 目标审查（Turn 7-13）

GRA处理目标审查相关的对话

### 6. 会话关闭（Turn 14-15）

SCA处理会话关闭

### 7. 生成摘要

SSA自动生成会话摘要，保存到 `session_summaries.json`

## 五、对话记录的数据结构

### goal_reviews.json 结构

```json
[
  {
    "patient_id": "patient_xyz789",
    "turn_index": 14,
    "chat_history": [
      {
        "role": "assistant",
        "content": "Hi Daniel! It's great to see you today..."
      },
      {
        "role": "user",
        "content": "I'm feeling pretty good..."
      },
      {
        "role": "assistant",
        "content": "It sounds like you're in a good mood..."
      }
    ]
  }
]
```

### 字段说明

- `patient_id`: 患者ID（由account_id映射而来）
- `turn_index`: 当前轮次（1-15）
- `chat_history`: 对话历史数组
  - `role`: "assistant" 或 "user"
  - `content`: 消息内容

## 六、注意事项

1. **数据持久化**: 对话记录保存在JSON文件中，服务重启后数据不会丢失
2. **数据隔离**: 每个patient_id的数据完全独立
3. **实时更新**: 每次发送消息后，记录会立即更新
4. **完整历史**: `goal_reviews.json` 包含所有代理的完整对话历史
5. **代理特定记录**: 各代理服务还维护自己的会话记录

## 七、故障排查

### 问题1：看不到对话记录

**检查**:
1. 确认MAS服务已启动
2. 检查JSON文件是否存在
3. 确认patient_id是否正确

### 问题2：对话记录不完整

**检查**:
1. 查看OA服务的日志
2. 检查 `goal_reviews.json` 文件
3. 确认所有代理服务正常运行

### 问题3：无法触发会话

**检查**:
1. 确认JWT token有效
2. 检查MAS服务健康状态：`GET /api/v1/mas/health`
3. 查看talkieai-server的日志
