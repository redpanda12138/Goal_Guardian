# MAS系统整合实施计划

## 项目概述

将MAS（Multi-Agent System）健康教练会话管理系统整合到talkieai-server后端，实现统一的API接口、数据管理和用户系统。

## 当前状态分析

### MAS系统现状
- ✅ 6个独立的FastAPI微服务
- ✅ 基于JSON文件的数据存储
- ✅ 使用OpenAI GPT-4.1进行AI对话
- ✅ 定时任务通过后台线程实现
- ✅ 独立的Docker Compose部署

### talkieai-server现状
- ✅ 单体FastAPI应用
- ✅ SQLite数据库（SQLAlchemy ORM）
- ✅ 完整的用户账户系统
- ✅ 支持ChatGPT和智谱AI
- ✅ 会话和消息管理系统
- ✅ 语言学习功能（翻译、语法、发音等）

### 数据模型设计

#### 1. 患者信息表（PatientInfoEntity）
```python
class PatientInfoEntity(Base):
    __tablename__ = "patient_info"
    
    id = Column(String(80), primary_key=True)
    account_id = Column(String(80), nullable=False)  # 关联到account表
    preferred_name = Column(String(100), nullable=True)
    hobbies = Column(Text, nullable=True)  # JSON数组字符串
    family = Column(Text, nullable=True)   # JSON数组字符串
    friends = Column(Text, nullable=True)   # JSON数组字符串
    travel = Column(Text, nullable=True)   # JSON数组字符串
    create_time = Column(DateTime, default=datetime.datetime.now)
    update_time = Column(DateTime, default=datetime.datetime.now, onupdate=datetime.datetime.now)
    
    # 索引
    account_id_index = Index("idx_patient_info_account_id", account_id)
```

## 技术选型

### 定时任务框架
**选择：APScheduler**
- 理由：轻量级，易于集成，支持多种调度方式
- 替代方案：Celery（如果未来需要分布式任务）

### AI服务集成
**使用现有的 `app.ai.chat_ai`**
- 统一AI服务调用
- 支持配置切换AI提供商

### 数据序列化
**JSON字段存储**
- 使用Text字段存储JSON字符串
- 在服务层进行序列化/反序列化

| AI服务调用失败 | 中 | 低 | 重试机制，降级处理，错误日志 |
