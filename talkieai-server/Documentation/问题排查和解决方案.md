# MAS Chat接入问题排查和解决方案

## 问题描述

1. Chat部分没有针对MAS进行处理和记录
2. Goal Review和Setting都没有获取到内容

## 问题分析

### 问题1：Chat部分没有MAS处理

**原因**：
- 前端创建的是CHAT类型session，不是MAS类型
- 需要创建MAS类型的session才能使用MAS对话

**已修复**：
- ✅ 修改了 `talkieai-uniapp/src/pages/index/index.vue` 的 `goChat()` 函数
- ✅ 现在点击"Chat with coach"会创建MAS类型的session

### 问题2：Goal Review和Setting没有内容

**可能原因**：
1. **数据尚未生成**：需要先提交会话笔记或完成健康教练会话
2. **patient_id映射问题**：account_id到patient_id的映射可能有问题
3. **MAS服务数据为空**：MMA服务中还没有该patient_id的数据

## 解决方案

### 步骤1：验证MAS服务运行

```bash
# 检查MAS服务健康状态
curl http://localhost:8097/api/v1/mas/health \
  -H "X-Token: {your_jwt_token}"

# 或直接检查各个服务
curl http://localhost:8001/health  # MMA
curl http://localhost:8002/health  # SOA
curl http://localhost:8006/health  # OA
```

### 步骤2：创建MAS会话并开始对话

1. **创建MAS会话**：
```bash
POST /api/v1/sessions/mas
Headers: X-Token: {jwt_token}
```

2. **获取问候语**（触发SOA）：
```bash
GET /api/v1/sessions/{session_id}/greeting
Headers: X-Token: {jwt_token}
```

3. **发送消息**：
```bash
POST /api/v1/sessions/{session_id}/chat
Headers: X-Token: {jwt_token}
Body: { "message": "我今天的能量水平是7分" }
```

### 步骤3：生成Goal Review和Setting数据

有两种方式：

#### 方式A：提交会话笔记（触发MMA提取）

```bash
POST /api/v1/mas/patients/notes
Headers: X-Token: {jwt_token}
Body: {
  "notes": [
    {
      "health_coach": "coach1",
      "date": "2024-01-01",
      "note": "患者今天状态良好，完成了本周的锻炼目标。目标：每天步行30分钟，每周5次。"
    }
  ]
}
```

这会触发MMA提取：
- 患者信息（preferred_name, hobbies, family, friends, travel）
- SMART目标

#### 方式B：完成一次健康教练会话

完成一次完整的MAS对话（15轮），系统会自动：
- 生成SMART目标（如果会话中设置了）
- 保存患者信息

### 步骤4：查看数据

#### 查看Goal Review

```bash
GET /api/v1/mas/patients/goals
Headers: X-Token: {jwt_token}
```

**如果返回空**：
- 检查是否已经提交过会话笔记
- 检查MMA服务的JSON文件：`mas/MMA/memory/weekly_smart_goals_mock.json`
- 确认patient_id是否正确

#### 查看Setting

```bash
GET /api/v1/mas/patients/info
Headers: X-Token: {jwt_token}
```

**如果返回空**：
- 检查是否已经提交过会话笔记
- 检查MMA服务的JSON文件：`mas/MMA/memory/session_notes_mock.json`
- 确认patient_id是否正确

## 常见问题

### Q1: Goal Review显示"No SMART goals found"

**原因**：
- 还没有提交会话笔记
- 或者会话笔记中没有提取到SMART目标

**解决**：
1. 提交包含SMART目标的会话笔记
2. 或完成一次健康教练会话，在会话中设置目标

### Q2: Setting显示"No patient information available"

**原因**：
- 还没有提交会话笔记
- MMA还没有提取患者信息

**解决**：
1. 提交会话笔记（包含患者信息）
2. 等待MMA处理（或手动触发）

### Q3: Chat对话没有记录

**原因**：
- session类型不是MAS
- MAS服务不可用
- patient_id映射失败

**解决**：
1. 确认创建的是MAS类型session
2. 检查MAS服务是否运行
3. 查看后端日志

### Q4: 创建MAS会话后没有问候语

**原因**：
- SOA服务不可用
- patient_id映射失败
- 等待时间不够

**解决**：
1. 检查SOA服务：`curl http://localhost:8002/health`
2. 增加等待时间（在get_session_greeting中）
3. 查看SOA日志

1. **启动MAS服务**
```bash
cd talkieai-server/mas
docker-compose up -d
```

2. **创建MAS会话**
```bash
POST /api/v1/sessions/mas
```

3. **获取问候语**
```bash
GET /api/v1/sessions/{session_id}/greeting
```

4. **发送几条消息**
```bash
POST /api/v1/sessions/{session_id}/chat
Body: { "message": "我今天的能量水平是7分" }
```

5. **提交会话笔记（生成数据）**
```bash
POST /api/v1/mas/patients/notes
Body: {
  "notes": [{
    "health_coach": "coach1",
    "date": "2024-01-01",
    "note": "患者Daniel今天状态良好。目标：每天步行30分钟，每周5次。"
  }]
}
```

6. **查看Goal Review和Setting**
```bash
GET /api/v1/mas/patients/goals
GET /api/v1/mas/patients/info
```

## 数据生成流程

```
提交会话笔记
    ↓
POST /api/v1/mas/patients/notes
    ↓
转发到MMA服务
    ↓
MMA提取信息：
  - 患者信息 → session_notes_mock.json
  - SMART目标 → weekly_smart_goals_mock.json
    ↓
前端可以获取数据：
  - GET /api/v1/mas/patients/info
  - GET /api/v1/mas/patients/goals
```
