# 方案二修改总结：通过Chat接入MAS

## 修改概述

本次修改将MAS系统集成到现有的chat对话系统中，用户可以在对话界面直接与MAS进行健康教练对话。同时，在practice页面显示用户的Goal Review和Setting信息。

## 一、后端修改

### 1.1 修改 `app/api/session_routes.py`

**文件**: `talkieai-server/app/api/session_routes.py`

**修改内容**:
- 在 `chat_api` 函数中添加MAS类型判断
- 如果session类型为MAS，调用MAS对话方法
- 保留原有的CHAT和TOPIC类型对话逻辑

**关键代码**:
```python
@router.post("/sessions/{session_id}/chat")
def chat_api(...):
    # 检查session类型
    if session.type == 'MAS':
        # MAS对话模式：转发到MAS服务
        return ApiResponse(
            data=chat_service.send_mas_session_message(session_id, dto, account_id)
        )
    else:
        # ========== 原有对话逻辑（保留） ==========
        return ApiResponse(
            data=chat_service.send_session_message(session_id, dto, account_id)
        )
```

### 1.2 修改 `app/services/chat_service.py`

**文件**: `talkieai-server/app/services/chat_service.py`

**新增方法**:
1. `send_mas_session_message()` - MAS对话消息发送
2. 修改 `get_session_greeting()` - 支持MAS类型的问候语

**关键功能**:
- `send_mas_session_message()`:
  - 获取patient_id（通过account_id映射）
  - 获取当前会话状态和turn_index
  - 根据turn_index判断当前代理（SOA/GRA/SCA）
  - 发送消息到对应的MAS服务
  - 获取助手回复并保存到数据库

- `get_session_greeting()`:
  - 如果是MAS类型，触发SOA开始会话
  - 获取第一条问候消息

**保留原有逻辑**:
- `send_session_message()` 方法完全保留，用于CHAT和TOPIC类型
- 所有原有方法都保留，只是添加了MAS分支

### 1.3 数据库支持

**Session类型**: `message_session.type` 字段现在支持：
- `CHAT` - 原有自由聊天（保留）
- `TOPIC` - 原有话题聊天（保留）
- `MAS` - 新增MAS健康教练对话

## 三、使用流程

### 3.1 创建MAS会话

需要创建一个type为"MAS"的session。可以通过以下方式：

1. **手动创建**（需要添加API）:
```python
# 在session_routes.py中添加
@router.post("/sessions/mas")
def create_mas_session(
    db: Session = Depends(get_db),
    account_id: str = Depends(get_current_account)
):
    """创建MAS健康教练会话"""
    chat_service = ChatService(db)
    return ApiResponse(
        data=chat_service.create_mas_session(account_id)
    )
```

2. **或修改现有创建接口支持MAS类型**

### 3.2 开始对话

1. 用户打开MAS类型的session
2. 如果没有消息，自动触发SOA开始会话
3. 用户发送消息，后端自动转发到MAS服务
4. MAS服务处理并返回回复
5. 前端显示对话内容

### 3.3 查看Goal Review和Setting

1. 用户进入practice页面
2. 自动加载Goal Review和Setting数据
3. 切换标签页查看不同信息

## 五、注意事项

1. **Session类型**: 确保创建的session类型为"MAS"
2. **MAS服务**: 确保MAS服务已启动（端口8001-8006）
3. **异步处理**: MAS服务调用使用异步，需要等待处理完成
4. **错误处理**: 如果MAS服务不可用，会返回错误信息
5. **数据同步**: MAS数据存储在JSON文件，不会自动同步到数据库

## 九、API端点总结

### MAS相关API（已实现）
- `POST /api/v1/mas/patients/notes` - 提交会话笔记
- `GET /api/v1/mas/patients/info` - 获取患者信息
- `GET /api/v1/mas/patients/goals` - 获取SMART目标
- `POST /api/v1/mas/sessions/trigger` - 触发会话
- `POST /api/v1/mas/sessions/message` - 发送消息
- `GET /api/v1/mas/sessions/current` - 获取当前会话状态
- `GET /api/v1/mas/sessions/history` - 获取对话历史
- `GET /api/v1/mas/sessions/summaries` - 获取会话摘要

### Chat相关API（保留原有）
- `POST /api/v1/sessions/{session_id}/chat` - 发送消息（支持MAS类型）
- `GET /api/v1/sessions/{session_id}/greeting` - 获取问候语（支持MAS类型）
