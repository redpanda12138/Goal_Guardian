## 使用说明

### 1. 环境配置

在 `.env` 文件中添加（可选，有默认值）：

```env
MAS_MMA_URL=http://localhost:8001
MAS_SOA_URL=http://localhost:8002
MAS_GRA_URL=http://localhost:8003
MAS_SCA_URL=http://localhost:8004
MAS_SSA_URL=http://localhost:8005
MAS_OA_URL=http://localhost:8006
```

### 2. 安装依赖

确保安装了 `httpx`（用于异步HTTP请求）：

```bash
pip install httpx
```

如果 `requirements.txt` 中没有，需要添加。

### 3. 数据库迁移

启动服务后，数据库会自动创建 `patient_mapping` 表。

### 4. 启动服务

1. 启动MAS服务（使用docker-compose或单独启动）
2. 启动talkieai-server
3. 验证服务健康状态：`GET /api/v1/mas/health`

## 架构说明

### 请求流程

```
客户端
  ↓ (JWT认证)
talkieai-server (网关层)
  ↓ (account_id → patient_id映射)
MAS服务 (独立服务)
  ↓ (处理请求)
返回响应
  ↓
talkieai-server (格式化响应)
  ↓
客户端
```

### 关键特性

1. **统一认证**: 所有请求通过talkieai-server的JWT认证
2. **自动映射**: account_id自动映射到patient_id
3. **透明转发**: 客户端无需知道MAS服务的具体地址
4. **错误处理**: 统一的错误处理和日志记录
5. **健康检查**: 可以检查所有MAS服务的健康状态

## 注意事项

1. **服务依赖**: 确保MAS服务在talkieai-server启动前已启动
2. **网络配置**: 如果MAS服务不在本地，需要配置正确的URL
3. **错误处理**: 当MAS服务不可用时，会返回错误信息
4. **数据存储**: MAS服务仍使用JSON文件存储，数据不会自动同步到数据库
5. **会话状态**: 获取当前会话状态的功能需要OA服务提供相应接口
